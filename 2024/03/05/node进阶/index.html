buffer

数组和对象都不能确定长度

但是buffer的是确定的

const { Buffer } = require('node:buffer');

// Creates a zero-filled Buffer of length 10.
const buf1 = Buffer.alloc(10);


// Creates a Buffer of length 10,
// filled with bytes which all have the value `1`.
const buf2 = Buffer.alloc(10, 1);

// Creates an uninitialized buffer of length 10.
// This is faster than calling Buffer.alloc() but the returned
// Buffer instance might contain old data that needs to be
// overwritten using fill(), write(), or other functions that fill the Buffer's
// contents.
const buf3 = Buffer.allocUnsafe(10);

// Creates a Buffer containing the bytes [1, 2, 3].
const buf4 = Buffer.from([1, 2, 3]);

// Creates a Buffer containing the bytes [1, 1, 1, 1] – the entries
// are all truncated using `(value & 255)` to fit into the range 0–255.
const buf5 = Buffer.from([257, 257.5, -255, '1']);

// Creates a Buffer containing the UTF-8-encoded bytes for the string 'tést':
// [0x74, 0xc3, 0xa9, 0x73, 0x74] (in hexadecimal notation)
// [116, 195, 169, 115, 116] (in decimal notation)
const buf6 = Buffer.from('tést');

// Creates a Buffer containing the Latin-1 bytes [0x74, 0xe9, 0x73, 0x74].
const buf7 = Buffer.from('tést', 'latin1');后面的也是一种特殊编码

dgram数据报

node:dgram 模块提供了 UDP 数据报套接字的实现

const dgram = require('node:dgram');
const server = dgram.createSocket('udp4');

server.on('error', (err) => {
  console.error(`server error:\n${err.stack}`);
  server.close();
});

server.on('message', (msg, rinfo) => {
  console.log(`server got: ${msg} from ${rinfo.address}:${rinfo.port}`);
});

server.on('listening', () => {
  const address = server.address();
  console.log(`server listening ${address.address}:${address.port}`);
});

server.bind(41234);
// Prints: server listening 0.0.0.0:41234

这是启动服务，那么如何接收消息呢？

使用send二进制和send字符串都可以

const dgram = require('node:dgram');
const { Buffer } = require('node:buffer');

const message = Buffer.from('Some bytes');
const client = dgram.createSocket('udp4');
client.send(message, 41234, 'localhost', (err) => {
  client.close();//这里是把客户端给关掉，建议是关掉
});

read的方法

和readfile的不同

const fs  = require('fs')
let buf =  Buffer.alloc(6)
fs.open('./1.txt',"r",(err,fd)=>{
    fs.read(fd,buf,3,3,3,(err,byteRead,buffer)=>{
        console.log(byteRead)
        console.log(buffer)
    })
})

上面代码的意思

  1. const fs = require('fs')：这行代码引入了 Node.js 的文件系统模块，使得我们可以使用其中提供的文件操作方法。
  2. let buf = Buffer.alloc(6)：这行代码创建了一个长度为 6 的 Buffer 对象，用于存储从文件中读取的数据。
  3. fs.open('./1.txt', "r", (err, fd) => {...})：这里使用 fs.open() 方法打开名为 1.txt 的文件以供读取。第二个参数 "r" 表示以只读模式打开文件。回调函数中的参数 fd 是文件描述符，用于后续读取操作。
  4. fs.read(fd, buf, 3, 3, 3, (err, bytesRead, buffer) => {...})：在打开文件后，使用 fs.read() 方法从文件中读取数据。具体参数含义如下：
    • fd：文件描述符，指向已打开的文件。
    • buf：要将数据读取到的 Buffer 对象。
    • 3：表示从文件的第 3 个字节位置开始读取数据。
    • 3：表示最多读取 3 个字节的数据。
    • 3：表示从 Buffer 对象的第 3 个位置开始写入读取的数据。
    • 回调函数参数 err 是错误对象（如果有错误发生），bytesRead 表示实际读取的字节数，buffer 包含了读取到的数据。

http2.0

这个提供的是http2的协议

var http2 = require('http2')
const fs = require('fs')
//一般http2.0之支持https，但是也可以支持http协议
//因为http+ssl = https
//公钥和私钥
const server =  http2.createServer({
    key:fs.readFileSync('rsa_private.key'),//安装公钥和私钥，本地证书直接去网上搜就可以
    cert:fs.readFileSync('cert.crt')
})
server.on('error',(err)=>{
    console.log(err)
})

server.on('stream',(stream,headers)=>{
    stream.respond(
        {
            'content-type': 'text/html',
            ':status':200
        }
    );
    stream.end('<h1>hello<h1>')
})

server.listen(3000)
